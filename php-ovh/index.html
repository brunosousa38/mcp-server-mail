Serveur MCP Mail (IMAP/SMTP)

Variante PHP pur, autonome — conçue pour un hébergement mutualisé (OVH, etc.).

Endpoint MCP

Le serveur répond en JSON-RPC 2.0 (transport MCP « Streamable HTTP », réponses JSON simples) sur : POST /mcp — ou POST /mcp.php si la réécriture d'URL est inactive.

Authentification

Toutes les requêtes exigent le token configuré dans config.local.php :

  • en-tête Authorization: Bearer <token> ;
  • ou paramètre d'URL ?token=<token> (pour les connecteurs qui ne permettent pas d'en-têtes personnalisés).

Outils exposés

  • list_folders — dossiers IMAP avec compteurs ;
  • list_emails — derniers emails d'un dossier ;
  • read_email — lecture complète d'un email (texte + pièces jointes) ;
  • search_emails — recherche (expéditeur, sujet, dates, non lus…) ;
  • send_email — envoi SMTP (texte brut, réponse dans un fil possible) ;
  • mark_email — marquer lu / non lu.

Installation

Copiez config.sample.php en config.local.php, renseignez le token (openssl rand -hex 32) et vos identifiants IMAP/SMTP, puis téléversez le dossier par FTP. Consultez le README du projet pour le détail.
